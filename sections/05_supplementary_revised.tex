\subsection*{A.4\quad Human validation and corpus-scale label reliability}
\phantomsection\label{supp:a4}

\paragraph{Validation design.}
Human review served two measurement roles. First, human--human agreement assessed whether trained reviewers could apply the finalized codebook consistently. The coding codebook check combined three validation rounds covering 658 task turns, with 643 turns after deduplication; the writing check used 210 reviewed conversations. Supplementary Table~\ref{tab:label_validation} reports the per-label metrics for this codebook-operability layer. Second, human--LLM audits compared final LLM-assisted corpus-scale labels with human-confirmed labels after item-level review. These audits covered user framing, user engagement and assistant support across the six corpus-by-task settings and are reported in Supplementary Table~\ref{tab:production_label_validation}.

\paragraph{Agreement metrics.}
Agreement is reported with per-label F1, Matthews correlation coefficient (MCC) and Gwet's AC1 because the labels are sparse, prevalence-imbalanced and sometimes multi-label~\cite{matthews1975comparison,gwet2008computing}. In the human--human codebook checks, the focal labels used in the main analyses showed acceptable or better agreement: Constructive user engagement reached F1=0.85 in coding and F1=0.89 in writing, and scaffolded support reached F1=0.87 in coding and F1=0.97 in writing (Supplementary Table~\ref{tab:label_validation}). Complete per-label F1, MCC, Gwet AC1 and support values are reported in Supplementary Table~\ref{tab:label_validation}.

\paragraph{Corpus-scale label audits.}
The consolidated human--LLM audits were stratified across the six corpus-by-task settings. The user-framing audit reviewed 450 first user turns, with 75 cases from each setting. Intentional-framing agreement reached F1=0.857, MCC=0.677 and Gwet AC1=0.671. The user-engagement audit reviewed 600 user turns, with 100 cases per setting; the constructive-versus-non-constructive distinction reached F1=0.922, MCC=0.840 and Gwet AC1=0.837. The assistant-side audit reviewed 180 assistant turns, with 30 cases per setting; scaffolded-support agreement reached F1=0.912, MCC=0.590 and Gwet AC1=0.791, and support-form F1 values ranged from 0.691 to 0.938 across M1--M6. Supplementary Table~\ref{tab:production_label_validation} reports the corresponding precision, recall, accuracy and positive-count fields.

\paragraph{Post-labelling WildChat review sample.}
After the WildChat corpus-scale labelling pass, an additional label-composition check reviewed 100 labelled turns from WildChat coding and 100 labelled turns from WildChat writing, split into 50 user turns and 50 assistant turns per task. Supplementary Table~\ref{tab:wildchat_post_labelling_review} reports the observed corpus-scale label composition of these samples.

\subsection*{A.5\quad Statistical estimands and model specifications}
\phantomsection\label{supp:a5}

\paragraph{Notation.}
This subsection gives the statistical notation and model specifications used for the main contrast and regression analyses. Let $i$ index conversations, $s$ task settings, $j$ adjacent assistant--user turn pairs and $g$ user-framing strata. $C_i$ is the count of Constructive user turns, $Y_i$ indicates whether conversation $i$ contains at least one Constructive user turn, $S_i$ indicates at least one scaffolded assistant turn in the conversation and $X_i$ denotes the setting-specific conversation-level covariates listed in Supplementary Table~\ref{tab:setting_model_covariates}. GLM specifications follow standard Poisson and logistic forms~\cite{mccullagh1989generalized,agresti2013categorical}.

\paragraph{Conversation-level models.}
The adjusted count model in Section~2.3 is fitted separately within each setting:
\begin{align*}
C_i \mid S_i, X_i &\sim \operatorname{Poisson}(\mu_i),\\
\log(\mu_i) &= \alpha_s + \beta_s S_i + \gamma_s^\top X_i .
\end{align*}
The reported Poisson count ratio is $\exp(\beta_s)$. The companion binary model is
\begin{align*}
\operatorname{logit}\{\Pr(Y_i=1 \mid S_i, X_i)\}
  = \alpha_s + \beta_s S_i + \gamma_s^\top X_i ,
\end{align*}
with $\exp(\beta_s)$ reported as the odds ratio. The user-framing stratified count model uses
\begin{align*}
\log(\mu_i) = \alpha_{s,g} + \beta_{s,g} S_i + \gamma_{s,g}^{\top} X_i^{(-g)},
\end{align*}
where $X_i^{(-g)}$ omits user framing because the model is fitted within stratum $g$.
These models support the Section~2.3 adjusted scaffolded-support results in Fig.~\ref{fig:s2_conversation_association}b,c and Table~\ref{tab:support_engagement_summary}; setting-specific covariates are listed in Supplementary Table~\ref{tab:setting_model_covariates}.

\paragraph{Rate and context checks.}
The Section~2.2 constructive-rate sensitivity is a grouped-binomial logistic model:
\begin{align*}
C_i &\sim \operatorname{Binomial}(T_i,\pi_i),\\
\operatorname{logit}(\pi_i) &=
\alpha + \delta_1 F_i + \delta_2 Q_i + \delta_3 L_i + \delta_4 D_i ,
\end{align*}
where $T_i$ is the number of user turns, $F_i$ user framing, $Q_i$ task ecology, $L_i$ conversation-length bucket and $D_i$ dataset fixed effects. The Section~2.3 offset-rate sensitivity uses the same Poisson mean model as above with
\begin{align*}
\log(\mu_i) = \log(T_i) + \alpha_s + \beta_s S_i + \gamma_s^\top X_i .
\end{align*}
Supplementary Tables~\ref{tab:fig3_offset_rate_sensitivity} and \ref{tab:constructive_rate_context_logit} report the estimates.
The grouped-binomial model is the Section~2.2 sensitivity for the conversation-length pattern in Fig.~\ref{fig:engagement_ecology}c, whereas the offset-rate Poisson model is the Section~2.3 sensitivity for the scaffolded-support count models in Fig.~\ref{fig:s2_conversation_association}b.

\paragraph{Adjacent-turn models.}
Let $Z_{ij}$ indicate whether the next user turn after assistant turn $j$ is Constructive, $U_{ij}$ encode the immediately preceding user state, $A_{ij}$ encode scaffolded support at assistant turn $j$, $M_{ij}$ encode support-form labels M1--M6 and $R_{ij}$ denote task, framing, turn-position and fixed-effect controls. The integrated adjacent-turn model is
\begin{align*}
\operatorname{logit}\{\Pr(Z_{ij}=1)\}
  = \alpha + \theta^\top U_{ij} + \beta A_{ij}
  + \phi^\top M_{ij} + \eta^\top (U_{ij}\times M_{ij}) + \rho^\top R_{ij}.
\end{align*}
The broad scaffolded-support model omits $M_{ij}$ and $U_{ij}\times M_{ij}$; model/source sensitivity checks replace dataset fixed effects in $R_{ij}$ with recoverable model or source-family indicators.
These models support the Section~2.5 adjacent-turn interpretation in Fig.~\ref{fig:temporal_coupling}d and Supplementary Tables~\ref{tab:integrated_regression}--\ref{tab:prior_state_support_form_interactions}; descriptive adjacent-turn lifts in Fig.~\ref{fig:temporal_coupling}a,b are summarized with bootstrap uncertainty in Supplementary Table~\ref{tab:key_contrast_significance}.

\paragraph{Uncertainty and multiplicity.}
Percentage-point contrasts use 1,000 conversation-level bootstrap resamples. Adjacent-turn contrasts use 1,000 conversation-cluster bootstrap resamples. Adjusted conversation-level GLMs use model-based standard errors; adjacent-turn regressions use conversation-cluster robust standard errors. Support-form bracket tests use Benjamini--Hochberg false-discovery-rate adjustment within each six-form comparison family~\cite{benjamini1995controlling}.

\subsection*{B.1\quad Privacy-preserving reporting}
\phantomsection\label{supp:b1}

\paragraph{Reporting boundary.}
The study analyses naturalistic public conversations, so reporting is restricted to aggregate statistics, de-identified examples, source-data summaries and validation metrics. The manuscript and release package do not redistribute bulk raw conversation logs, user identifiers, API credentials or linked user histories. Conversation and turn identifiers used in derived data are non-semantic internal identifiers rather than account identifiers. Operational examples are de-identified and usually paraphrased; Tables~\ref{tab:complete_coding_case} and \ref{tab:complete_writing_case} provide two illustrative case excerpts with personal details removed or generalized while preserving turn order and annotation decisions. The analyses do not link conversations into user histories or estimate longitudinal user-level trajectories; temporal claims are restricted to within-conversation ordering.
